/matches/<slug>¶

GET /matches/(string: slug)¶

Obtain complete details for a match.

Parameters:
  • slug (string) – unique slug of the match
Response JSON Object:
 
  • id (int) – unique id of the match
  • title (string) – title of the match
  • short_description (string) – short description of the match
  • long_description (string) – long description of the match. This is Markdown enabled, but will be empty unless the match has already started
  • start_date (date) – start date of the match
  • end_date (date) – end date of the match
  • min_team (int) – minimum number of participants for each team
  • max_team (int) – maximum number of participants for each team
  • slug (string) – unique slug of the match
Status Codes:
  • 200 OK – when record was found; returns match details
  • 404 Not Found – when record was not found or it was deleted

Example request:

GET /matches/practice-match HTTP/1.1
Host: example.com
Accept: application/json
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "id": 42,
    "title": "Practice match",
    "short_description": "This is a practice match",
    "long_description": "This is a practice match. LONG DESC",
    "start_date": "2016-04-23T18:25:43.511Z",
    "end_date": "2016-05-15T18:25:43.511Z",
    "min_team": 2,
    "max_team": 4,
    "slug": "practice-match"
}
PUT /matches/(string: slug)¶

Modify the details of a match.

This endpoint requires the admin role, therefore the server will check the supplied JWT and the stored role.

Parameters:
  • slug (string) – unique slug of the match
Request JSON Object:
 
  • token (string) – JWT of the active user session
  • title (string) – title of the match
  • short_description (string) – short description of the match
  • long_description (string) – long description of the match (Markdown enabled)
  • start_date (date) – start date of the match
  • end_date (date) – end date of the match
  • min_team (int) – minimum number of participants for each team
  • max_team (int) – maximum number of participants for each team
  • slug (string) – unique slug for the match; can be created automatically if it is not provided
  • is_visible (boolean) – whether the match is visible the public
Response JSON Object:
 
  • id (int) – unique id of the match
  • title (string) – title of the match
  • short_description (string) – short description of the match
  • long_description (string) – long description of the match (Markdown enabled)
  • start_date (date) – start date of the match
  • end_date (date) – end date of the match
  • min_team (int) – minimum number of participants for each team
  • max_team (int) – maximum number of participants for each team
  • slug (string) – unique slug for the match
  • is_visible (boolean) – whether the match is visible the public
Status Codes:
  • 200 OK – when record was found; returns updated match details
  • 401 Unauthorized – when the user does not have the required admin role
  • 404 Not Found – when record was not found or it was deleted

Example request:

PUT /matches/practice-match HTTP/1.1
Host: example.com
Accept: application/json
Content-Type: application/json

{
    "token": "JWT_TOKEN",

    "title": "Updated Practice match",
    "short_description": "This is a practice match, updated",
    "long_description": "This is the long description, updated"
    "start_date": "2016-04-23T18:25:43.511Z",
    "end_date": "2016-05-15T18:25:43.511Z",
    "min_team": 1,
    "max_team": 1,
    "is_visible": false
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "id": 42,
    "title": "Updated Practice match",
    "short_description": "This is a practice match, Updated",
    "long_description": "This is the long description, updated"
    "start_date": "2016-04-23T18:25:43.511Z",
    "end_date": "2016-05-15T18:25:43.511Z",
    "min_team": 1,
    "max_team": 1,
    "is_visible": false
}
