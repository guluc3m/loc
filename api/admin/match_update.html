/admin/matches/<id>/update¶

POST /matches/(int: id)/update¶

Modify the details of a match.

Parameters:
  • id (int) – unique id of the match
Request JSON Object:
 
  • token (string) – JWT token of the active user session
  • title (string) – title of the match
  • description (string) – long description of the match (Markdown enabled)
  • start_date (date) – start date of the match
  • end_date (date) – end date of the match
  • min_team (int) – minimum number of participants for each team
  • max_team (int) – maximum number of participants for each team
  • visible (boolean) – whether the match is visible to the public or not
  • title – title of the match
  • description – long description of the match (Markdown enabled)
  • start_date – start date of the match
  • end_date – end date of the match
  • min_team – minimum number of participants for each team
  • max_team – maximum number of participants for each team
  • visible – whether the match is visible to the public or not
Response JSON Object:
 
  • id (int) – unique id of the match
Status Codes:
  • 200 OK – when record was found; returns updated match details
  • 401 Unauthorized – when the user does not have the required admin role
  • 404 Not Found – when record was not found or it was deleted

Example request:

POST /admin/matches/42/update HTTP/1.1
Host: example.com
Accept: application/json
Content-Type: application/json

{
    "token": "JWT_TOKEN",

    "title": "UPDATED Practice match",
    "description": "This is a practice match, UPDATED",
    "start_date": "2016-04-23T18:25:43.511Z",
    "end_date": "2016-05-15T18:25:43.511Z",
    "min_team": 1,
    "max_team": 1,
    "visible": true
}

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "id": 42,

    "title": "UPDATED Practice match",
